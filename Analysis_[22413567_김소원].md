# 1. Introduction

본 문서는 "원 스텝 천 리"의 구현을 위한 세부 분석 내용을 소개한다. 사용자의 시스템 사용 시나리오를 다루는 Use case analysis, 시스템에 필요한 클래스를 분석하는 Domain analysis, 그리고 실제 화면 구성 프로토타입을 보여주는 User Interface prototype을 통해 시스템의 전체적인 윤곽을 구체화한다. 

## 1.1. Summary

현대 사회에서, 다수 사람들은 과제나 업무를 앞두고 시작하는 데에 어려움을 겪거나 미루는 습관을 가지고 있다. 특히 주의력 결핍 및 과잉행동장애(ADHD)를 겪는 사람들의 경우 이러한 시간 관리와 집중력 유지에 더욱 취약하여 업무를 끝까지 완료하는 데에 큰 불편함을 겪는 경우가 많다. 그러나 시중에 나와 있는 대부분의 학습 관리 프로그램들은 달성한 ‘과제의 양’보다, ‘공부 시간’을 중시하기에 타이머만 켜 두고 집중력이 저하된 채 의미 없는 시간을 보내게 되는 경우가 많았다. 따라서 이러한 기존 프로그램들의 한계점과 시간 관리의 불편함을 해소하고, 과제나 업무를 미루지 않고 시작할 수 있게 돕는 프로그램을 고안하게 되었다.
<br />

## 1.2. Business Goals

해당 시스템의 주된 목적은 사용자가 과제를 미루지 않도록 심리적 장벽을 낮추고, 실질적인 과제 달성률에 집중할 수 있는 환경을 제공하는 것이다. 막연하고 큰 과제를 작은 단위로 나누는 ‘Eat that frog’ 시간 관리법으로 사용자의 부담감을 낮추고 과제를 독려한다. 또한, 오랜 시간동안 의미없이 과제를 붙잡고 있는 것을 지양하기 위해 25분간 집중하고 5분간 휴식하는 ‘Pomodoro Technique(포모도로 기법)’ 타이머 기능을 지원해, ADHD를 겪는 사람들과 미루는 습관을 가진 사용자들이 업무 효율성과 과제 달성률을 향상할 수 있도록 한다.
<br />

# 3. Domain analysis
다음은 시스템의 데이터 및 내부 구조를 파악하기 위한 Domain Analysis이다. 본 장에서는 시스템 구동에 필수적인 핵심 객체(과제, 타이머, 사용자 등)를 식별하고, 이들 간의 관계를 도식화하여 프로그램의 구조를 명확하게 정의한다.
<br />

## 3.1. User 

사용자의 기본 정보를 담는 클래스다. 회원가입 시 입력되는 아이디, 비밀번호, 닉네임 등의 속성을 가지며 시스템 내에서 사용자를 식별하는 기준이 된다.


## 3.2. Task 
사용자의 목표와 메인과제 , 하위 과제, 공동 과제 데이터를 관리하는 클래스다. 메인 과제와 하위 과제 구분 여부, 메인 과제와 하위 과제의 내용, 완료 상태, 과제의 부모-자식 관계, 과제의 소유자 정보, 공동 과제 여부, 공동 과제 원본과 복사본을 기록하는 ID, 과제를 공유받은 친구(user) 객체 정보를 저장한다.


## 3.3. Friend 

사용자 간의 친구 관계 데이터를 저장하는 클래스이다. 친구를 추가한 사용자와 친구로 추가된 사용자의 조합을 식별하는 ID, 현재 관계를 구분하는 식별자에 관한 정보를 저장한다.


## 3.4. Stat 

사용자의 일일 통계 데이터를 기록하는 클래스이다. 해당 통계의 user ID, 통계가 기록된 날짜, 당일 누적된 타이머 시간, 연속 달성일, 당일의 과제 달성률에 대한 정보를 저장한다. 


## 3.5. UserService 

Register, Login, Logout 등의 인증 역할을 수행하는 클래스다. 시스템을 사용하려는 회원의 정보를 데이터베이스와 대조하여 회원가입 / 로그인 성공 여부를 판별하고 관리한다.


## 3.6. TaskService 

게시물(과제)의 전반적인 기능을 관리하는  클래스이다. Add Task(과제 등록), Update Task(과제 수정), Delete Task(과제 삭제), Toggle Task(과제 상태 변경) 기능을 수행한다. 공동 과제의 경우 Add Shared Task(과제를 복제하고 친구들의 정보와 동기화하는 기능)과 개인의 과제 목록을 조회하는 기능도 포함한다.


## 3.7. FriendService 

친구 관계를 관리하는 클래스이다. Add Friend(다른 사용자의 아이디를 검색하여 내 친구 목록에 추가하는 기능), 특정 유저를 이미 친구로 추가했는지 중복 여부를 검사하는 기능을 수행한다.


## 3.8. StatService 

사용자의 통계와 랭킹을 계산하는 클래스이다. 타이머가 종료되었을 때 사용자의 일일 통계에 시간을 더하는 기능(해당 날짜의 타이머 기록이 이미 존재한다면 기존 시간에 더하고, 첫 기록이라면 새 통계 기록을 생성한 뒤 시간을 더한다.), 최근 7일간의 과제 달성률 평균과 집중 시간 평균치를 계산하고 조회하는 기능(오늘로부터 6일 전의 날짜 범위를 계산한 뒤, 해당 기간 동안의 통계 기록들을 날짜 오름차순으로 반환한다.), 당일 과제 달성률을 계산하고, 친구들의 기록과 비교해 랭킹을 산출하는 기능(등록된 과제 중 완료된 과제 개수를 세어 달성률을 계산한 후, 내림차순 정렬해 반환한다.) 등이 포함된다.

# 4. User Interface prototype

본 장에서는 요구사항 분석 단계를 거쳐 구체화된 시스템의 UI 프로토타입을 소개한다. 사용자가 시스템과 직접 상호작용하게 될 주요 화면의 구성 요소와 화면 간의 전환 시나리오를 제시하여, 추후 개발될 프로그램의 형태와 사용자 인터페이스(UI) 구조를 명확히 이해하는 것을 목적으로 삼는다.
<br />

## 4.1. Register / Login


처음 System을 실행했을 때 사용자가 마주하는 창으로, 회원가입으로 전환 / 로그인으로 전환 버튼을 눌러 원하는 기능을 사용할 수 있다. 
- 아이디와 비밀번호, 닉네임을 입력하고 ‘가입하기’ 버튼을 누르면 로그인 창으로 자동 전환된다.
- 이후 ‘로그인’ 버튼을 누르면 기능을 이용할 수 있는 화면으로 전환된다.
- 아이디는 중복될 수 없으며, 공백 기호를 아이디와 비밀번호, 닉네임으로 사용할 수 없다.


## 4.2. Main Screen
성공적으로 로그인하면 Main Screen으로 전환된다. 모든 일정 관리 기능은 Main Screen에서 사용할 수 있다. 화면 좌측과 중앙엔 과제를 등록하고 과제 목록을 보거나 등록한 과제를 수정, 삭제하는 대시보드, 타이머 기능을 이용하는 대시보드가 위치하며 화면 좌측의 ‘나의 기록’, ‘친구’ 버튼을 눌러 적절한 기능의 대시보드를 불러올 수 있다. 화면 상단 좌측엔 로고가, 우측에는 로그아웃 버튼이 위치한다.


### 4.2.1. Logout
Main Screen 우측 상단의 ‘로그아웃’ 버튼을 눌러 처리한다.


### 4.2.2. Task Dashboard
과제를 등록하고 관리하는 기능을 제공한다. 
- 메인 과제를 등록할 때 ‘혼자하기’ 또는 ‘친구와 함께하기’ 옵션을 선택해 일반 과제로 추가할지, 공동 과제로 추가할지 정할 수 있다. 
- 공동 과제는 메인 과제명 옆에 ‘[공동]’ 이라는 키워드가 붙지만, 일반 과제는 아무 키워드도 붙지 않는다.
- 하위 과제를 추가하기 위해서는 먼저 메인 과제를 입력해야 하며, 등록한 메인 과제는 ‘과제 목록’ 대시보드에서 확인할 수 있다. 
- 메인 과제명 옆의 ‘추가’ 버튼을 눌러 하위 과제를 등록할 수 있으며, 각 과제명 옆의 ‘수정’, ‘삭제’ 버튼을 누르면 그에 해당하는 기능을 사용할 수 있다. 
- 과제를 수정할 때 빈 칸을 입력할 수 없으며, 메인 과제를 삭제하면 해당 과제와 연결된 모든 하위 과제가 삭제된다. 
- 하위 과제는 다른 하위 과제와 독립적인 성격을 띠며, 한 하위 과제의 수정과 삭제가 다른 하위 과제에 영향을 미치지 않는다.


### 4.2.3. Timer Dashboard
25분의 타이머 기능을 사용할 수 있다. 
- ‘시작’ 버튼을 누르면 카운트다운이 시작되며 화면의 숫자가 줄어든다. 
- 25분이 지나면 타이머 시간이 끝났다는 내용의 메시지가 발송되며, 사용자가 ‘확인’ 버튼을 누를 경우 자동으로 5분의 휴식 시간을 카운트다운한다. 


### 4.2.4. Stat Dashboard
Main Screen 우측 상단의 ‘나의 기록’ 버튼을 눌러 개인의 통계 기록을 볼 수 있다. 
- 연속 달성일, 최근 7일간의 평균 과제 달성률, 당일 과제 달성률을 확인할 수 있으며 최근 7일간의 집중 시간을 그래프 형식으로 시각화하여 나타낸다. 
- 사용자가 과제를 완료하고 타이머 기능을 이용하는 등의 변화가 생기면 ‘당일 과제 달성률’과 ‘집중 시간 그래프’에 즉각 반영된다.


### 4.2.5. Friend Dashboard
Main Screen 우측 상단의 ‘친구’ 버튼을 눌러 랭킹을 확인하거나 친구를 추가하고, 친구 목록을 확인할 수 있다.
- Stat Dashboard의 ‘당일 과제 달성률’과 ‘집중 시간 그래프’와 마찬가지로 사용자의 과제 달성률에 따라 랭킹이 수시로 바뀐다. 
- 친구를 추가할 땐 아이디를 이용하며, 사용자 A가 사용자 B를 친구로 추가할 경우 사용자 B가 A를 친구로 추가하지 않아도 자동으로 친구 목록에 반영된다. 
- 친구관계를 삭제할 때엔 ‘아이디(닉네임)’ 옆의 ‘삭제’ 버튼을 눌러 처리한다.


# 5. Glossary
아래는 해당 보고서에 사용된 용어들의 구체적인 정의이다.


| Terms | Description |
| --- | --- | 
| 리더 (Leader) | 친구가 한 명 이상인 사용자. 공동 과제를 추가할 수 있는 권한을 부여받는다. | 
| 메인 과제 | 궁극적으로 달성하고자 하는 과제를 의미한다. 체크박스 형식으로 표시되는 하위 과제와 달리, 메인 과제는 체크박스 형식이 아니라 줄글 형식이며 하위 과제를 모두 완수해야만 해당 과제가 완료된 것으로 처리된다.  | 
| 하위 과제 | 궁극적으로 달성해야 하는 메인 과제를 세분화한 것으로, 체크박스 형식으로 나타낸다. 예를 들어, ‘보고서 10p 작성’이 목표라면 ‘노트북 켜기’, ‘관련 논문 찾아보기’, ‘초안 작성하기’ 등이 하위 과제가 될 수 있다. | 
| 공동 과제 | 리더 (Leader) 가 과제를 추가한 후 친구들과 공유할 수 있는 과제를 의미한다. 과제 추가 후 공동 과제로 등록하면, 서버가 해당 사용자의 친구 목록을 조회한 후 각 친구들에게 해당 과제를 자동 복사 후 추가한다. 이렇게 저장된 공동 과제는 개개인이 자신의 화면에서 독립적으로 관리할 수 있다. |
| 과제 달성률 | 오늘 생성된 과제 수와 완료 수의 비율이다. '달성된 하위 과제의 개수 / 생성된 하위 과제의 개수' 로 계산한다. | 
| 연속 달성일 | 연속으로 과제 달성률이 100%인 날짜 수를 의미한다. 예를 들어, 5일간 과제 달성률이 100%였다면 해당 사용자의 연속 달성일은 5일이다. 연속 달성일을 이어가다가 과제 달성률이 100% 미만이 되면 연속 달성일은 0일로 리셋된다. |
| 연속 달성 격려 메시지 | 연속 달성일이 이틀 이상인 사람에게 발송되는 푸쉬 알림으로, 사용자의 성취감을 고취시키기 위해 고안되었다. |
| 푸시 알림 | 웹페이지에서 사용자에게 보내는 알림이다. 당일 과제 달성률을 보고하거나 연속 달성 격려 메시지를 보낼 때 사용된다. |
| 과제 리마인더 | 한 시간마다 미완료 상태인 하위 과제의 개수를 리마인드 시키며, 과제 수행을 촉구하는 푸쉬 알림이다. |
| 랭킹 | 친구들과 나의 과제 달성률을 계산해 내림차순 정렬 후 매긴 순위로, 친구를 추가한 사용자들이 이용할 수 있는 기능이다. |
